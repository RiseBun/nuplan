## 🔧 脚本功能详解

### 自动数据准备

脚本会按顺序检查：

```bash
# 1. 检查训练索引
if [ ! -f "indices/consistency_train.jsonl" ]; then
    # 2. 检查原始数据
    if [ ! -d "/mnt/cpfs/prediction/lipeinan/nuplan_data/mini_set" ]; then
        echo "❌ 数据不存在，请先准备 nuPlan 数据！"
        exit 1
    fi
    
    # 3. 生成训练数据（如果不存在）
    if [ ! -d "critic_training_data" ]; then
        python generate_critic_training_data.py \
            --num-scenes 100 \
            --samples-per-scene 5
    fi
    
    # 4. 计算标签（如果不存在）
    if [ ! -d "critic_training_data_labeled" ]; then
        python compute_training_labels.py
    fi
    
    # 5. 构建索引
    python build_critic_index.py
fi
```

**智能检测**:
- ✅ 如果数据已存在，跳过生成
- ✅ 如果部分数据存在，只生成缺失的部分
- ✅ 全部自动化，无需手动干预

### 多卡训练

脚本自动处理：

```bash
# 自动检测 GPU 数量
DETECTED_GPUS=$(nvidia-smi -L | wc -l)
NPROC_PER_NODE=$DETECTED_GPUS

# 启动分布式训练
python -m torch.distributed.run \
    --nnodes=1 \
    --nproc_per_node=$NPROC_PER_NODE \
    --node_rank=0 \
    --rdzv_backend=c10d \
    --rdzv_endpoint=localhost:29500 \
    train.py \
    --config configs/train_consistency_mini.py \
    --epochs 50 \
    --batch-size 16 \
    --work-dir work_dirs/critic_full
```
