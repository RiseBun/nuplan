# nuPlan 闭环评估使用指南

## 📋 概述

本系统用于证明 **Consistency Critic 与 nuPlan 闭环性能的相关性 > FID/FVD**，这是论文的核心贡献点。

### 实验目标

```
证明:
  Spearman(Critic分数, 闭环性能) > Spearman(FID分数, 闭环性能)
  Spearman(Critic分数, 闭环性能) > Spearman(FVD分数, 闭环性能)

即:
  我们的 visual-action metric 比传统生成质量指标
  更能反映 driving usefulness
```

## 📊 实验设计

### 流程

```
For each scene:
  1. 生成 10 个候选轨迹（不同噪声级别）
     ├─ 候选 0: 真实轨迹（无噪声）
     ├─ 候选 1-4: 低噪声
     ├─ 候选 5-7: 中噪声
     └─ 候选 8-9: 高噪声
  
  2. 三种评分方式
     ├─ Critic 评分（越高越好）
     ├─ FID 评分（越低越好）
     └─ FVD 评分（越低越好）
  
  3. 选择 top-1 轨迹
     ├─ Best by Critic
     ├─ Best by FID
     └─ Best by FVD
  
  4. 计算闭环指标
     ├─ NC (No At-Fault Collision)
     ├─ TTC (Time To Collision)
     ├─ Comfort
     ├─ Progress
     └─ Overall
  
  5. 记录结果

After all scenes:
  6. 计算 Spearman/Kendall 相关性
     ├─ Critic vs 闭环指标
     ├─ FID vs 闭环指标
     └─ FVD vs 闭环指标
  
  7. 计算 Gain
     ├─ Gain vs FID = Spearman_Critic - Spearman_FID
     └─ Gain vs FVD = Spearman_Critic - Spearman_FVD
  
  8. 可视化
```

## 🔍 核心指标

### 闭环指标

| 指标 | 全称 | 含义 | 范围 | 越高越好 |
|------|------|------|------|---------|
| **NC** | No At-Fault Collision | 无责任碰撞率 | [0, 1] | ✅ |
| **TTC** | Time To Collision | 碰撞时间（秒） | [0, ∞) | ✅ |
| **Comfort** | 舒适度 | 加速度/转向平滑度 | [0, 1] | ✅ |
| **Progress** | 进度 | 前进距离 | [0, 1] | ✅ |
| **Overall** | 综合分数 | 加权平均 | [0, 1] | ✅ |

### 相关性指标

| 指标 | 含义 | 目标 |
|------|------|------|
| **Spearman ρ** | 排名相关性 | Critic > FID/FVD |
| **Kendall τ** | 排名一致性 | Critic > FID/FVD |
| **p-value** | 统计显著性 | < 0.05 |
| **Gain** | 提升幅度 | > 0 (越大越好) |

def compute_confidence_interval(correlations, n_bootstrap=1000):
    """计算 95% 置信区间"""
    bootstrap_results = []
    
    for _ in range(n_bootstrap):
        # 有放回采样
        sampled = np.random.choice(correlations, size=len(correlations), replace=True)
        bootstrap_results.append(np.mean(sampled))
    
    ci_lower = np.percentile(bootstrap_results, 2.5)
    ci_upper = np.percentile(bootstrap_results, 97.5)
    
    return ci_lower, ci_upper
